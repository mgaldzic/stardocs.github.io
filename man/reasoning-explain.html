stardog reasoning explain Manual Page

NAME


stardog reasoning explain —Provide the explanation for an inference or inconsistency.

SYNOPSIS


stardog reasoning explain [(-f <format> | --format <format>)] [(-i | --inconsistency)] [(-o <format> | --output-format <format>)] [(-p <password> | --passwd <password>)] [(-P | --ask-password)] [(-r <level> | --reasoning <level>)] [(-u <username> | --username <username>)] [(-v | --verbose)] [--] <database> <file>...

OPTIONS


-f <format>, --format <format>
Input data format: NTRIPLES, RDF/XML, TURTLE, TRIG, TRIX, N3, NQUADS
-i, --inconsistency
Explain inconsistency of the database
-o <format>, --output-format <format>
Output format for the explanation: [TEXT, TREE, NTRIPLES, RDF/XML, TURTLE, TRIG, TRIX, N3, NQUADS]. The default format is 'TEXT' where theexplanation is rendered in a concise, human-readable format.
-p <password>, --passwd <password>
Password.
-P, --ask-password
Prompt for password.
-r <level>, --reasoning <level>
The reasoning level to use for query. Can be one of [NONE, QL, RL, EL, SL, DL, RDFS]
-u <username>, --username <username>
User name.
-v, --verbose
Verbose mode prints the axiom before the explanation
--
This option can be used to separate command-line options from the list of argument, (useful when arguments might be mistaken for command-line options
<database> <file>
The name of the database or the full connection string of the database to connect to. If only the name is provided, the default server URL will be pre-pended to the name of the database in order to construct the connection string. Connection parameters such as ';reasoning=QL' can be included in the provided database name. Connection parameters specified like this can be overridden by specific options on the command. The default server URL will be read from the JVM argument 'stardog.default.cli.server'. If the JVM argument is not set, the default value 'snarl://localhost:5820' is used. If the server URL has no explicit port value, the default port value '5820' is used. To use a secure connection, you should specify the full connection string and postfix 's' to the protocol, e.g. snarls or https.

DISCUSSION


Returns an explanation for the inference in the given file or for the inconsistency of the database. If provided, the input file should contain a single axiom. The explanation output is the minimal set of asserted statements in the database that are needed for the inference to hold. The statements in the explanatino can be rendered in any RDF format but by default they will be rendered in a concise, human-readable, text format. There is also a proof tree format where the statements are rendered in hierarchical structure. Nodes in the proof tree are grouped under inferred statements that make it easier to understand the explanation.
