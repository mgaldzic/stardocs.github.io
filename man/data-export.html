stardog data export Manual Page

NAME


stardog data export —Exports the contents of a database in RDF.

SYNOPSIS


stardog data export [ {-f | --format} <RDF Format> ] [ {-g | --named-graph} <export graph> ] [ {-p | --passwd} <password> ] [ {-P | --ask-password} ] [ {-u | --username} <username> ] [--] <dbname> <export file>...

OPTIONS


-f <RDF Format>, --format <RDF Format>
RDF Format [NTRIPLES, RDF/XML, TURTLE, TRIG, TRIX, N3, NQUADS] for the exported data.
-g <export graph>, --named-graph <export graph>
Named graph to export.
-p <password>, --passwd <password>
Password.
-P, --ask-password
Prompt for password.
-u <username>, --username <username>
User name.
--
This option can be used to separate command-line options from the list of argument, (useful when arguments might be mistaken for command-line options
<dbname> <export file>
The name of the database or the full connection string of the database to connect to. If only the name is provided, the default server URL will be pre-pended to the name of the database in order to construct the connection string. Connection parameters such as ';reasoning=QL' can be included in the provided database name. Connection parameters specified like this can be overridden by specific options on the command. The default server URL will be read from the JVM argument 'stardog.default.cli.server'. If the JVM argument is not set, the default value 'snarl://localhost:5820' is used. If the server URL has no explicit port value, the default port value '5820' is used. To use a secure connection, you should specify the full connection string and postfix 's' to the protocol, e.g. snarls or https. The second argument is the file to write the output to; if not specified, the output is written to STDOUT.

DISCUSSION


Exports the database in the specified format to the specified output file, else to STDOUT. If no format is specified, the format is determined from the output file name; otherwise, TURTLE is the default. If one of TRIG, TRIX, or NQUADS is specified, then all named graphs data will be preserved on export; otherwise, only the default graph will be exported. If named-graph is specified, only the contents of that named graph will be exported.

EXAMPLES


Export the database 'myDb' as NTRIPLES:

    $ stardog data export --format NTRIPLES myDb

Export a named graph of the database 'myDb' as TRIG:

    $ stardog data export --format TRIG --named-graph http://mycontext.org/theContextid myDb

Export the database 'myDb' to a file in TURTLE:

    $ stardog data export myDb export.ttl
